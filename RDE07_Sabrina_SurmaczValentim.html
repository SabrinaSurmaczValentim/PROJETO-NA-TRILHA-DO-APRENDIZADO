BRANCHES

  São ramificações de um projeto de software ou documento e tem como objetivo o desenvolvimento paralelo de:

  • Novas funcionalidades;
  • Correção de problemas;
  • Alteração de código.

TIPOS DE BRANCHES

  Quando temos um projeto e o projeto vai ser colocado em um repositório, podemos colocar ele na Branch Master ou na Branch Secundária.

  Quando colocamos um projeto no repositório, provavelmente ele vai ficar na Branch Master. Posso realizar alterações no projeto e realizo o commit.

  Porém, imagine que a equipe do Mercado Livre está realizando alterações na Branch Master e realizando o commit, enquanto os usuários estão fazendo as compras, há muita chance de acontecer algum problema. A forma correta é fazer alterações em Brachs secundários (não vão fazer alterações na Master) e não no Branch Master. Quando há certeza que está tudo certo pode fazer o merge, ou seja, a mesclagem das Braches Secundárias com a Branch Master.
  OBSERVAÇÃO: A Branch Master também pode ser chamada de Brach Main.

HOSPEDAGEM

  Hospedagem de um site - para que qualquer usuário com acesso à internet possa acessá-lo. Geralmente é necessário pagar pelo serviço de hospedagem, porém, existe um recurso do GitHub que é gratuito.
  O GitHub Pages oferece hospedagem gratuita de sites. Basta criar uma conta e um repositório público e a URL será:
https: / /nomeusuario.github.io /repositorio

  No site GitHub...

  Clicar em Settings...

  Aí rola a página até encontrar GitHub Pages...

  OSERVAÇÃO: as vezes demora para carregar o site quando clica, basta esperar.
  Se demorar demais para atualizar o site, fica página não encontrada, basta ir no GitHub Desktop, clica no repositório, abre no VSCode, faz alguma alteração, faz o commit e após push origin. Assim, agora irá funcionar.

  Se um repositório está privado, não tem como fazer a hospedagem, então precisa alterar a licença...
